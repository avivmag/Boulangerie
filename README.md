קינוחים
אפליקציה נקייה למסעדה: לקוח סורק QR, ממלא שם מלא, טלפון ומייל, מאשר דיוור אם הוא רוצה, ומקבל קוד חד-פעמי ל-2 קינוחים במתנה.

הפעלה
npm start
פותחים בדפדפן:

http://localhost:3000
קובץ מעקב לאקסל
כל הרשמה נשמרת כאן:

data/leads.csv
אפשר לפתוח את הקובץ ישירות באקסל. יש גם הורדה דרך:

http://localhost:3000/api/export
מניעת כפילויות
המערכת לא מאפשרת לממש שוב אם אותו טלפון או אותו מייל כבר קיימים. הנתונים הפנימיים נשמרים כאן:

data/claims.json
התאמה למסעדה
כדי להחליף לוגו, אפשר להוסיף קובץ תמונה לתיקיית public ולעדכן את אזור brand-mark ב-public/index.html.

העלאה לאינטרנט
האפליקציה כוללת שרת ששומר הרשמות ומונע כפילויות, ולכן לא מתאימה ל-GitHub Pages בלבד.

הדרך הפשוטה:

מעלים את כל התיקייה ל-GitHub Repository.
פותחים חשבון ב-Render.
בוחרים New Web Service.
מחברים את ה-Repository.
Render יזהה את render.yaml ויריץ את האפליקציה.
אחרי הפריסה מקבלים כתובת ציבורית שאפשר להפוך ל-QR.
הערה חשובה: שמירה לקובץ CSV מתאימה לפיילוט. לקמפיין פעיל מומלץ לחבר Google Sheets או מסד נתונים כדי שהלידים יהיו שמורים גם אחרי פריסות ועדכונים.

חיבור ל-Google Sheets
יוצרים Google Sheet חדש.
בתפריט בוחרים Extensions ואז Apps Script.
מדביקים את הקוד מהקובץ google-apps-script.js.
לוחצים Deploy ואז New deployment.
בוחרים Web app.
תחת Execute as בוחרים Me.
תחת Who has access בוחרים Anyone.
לוחצים Deploy ומאשרים הרשאות.
מעתיקים את כתובת ה-Web app שמסתיימת ב-/exec.
ב-Render מוסיפים Environment Variable בשם GOOGLE_SHEETS_WEBHOOK_URL עם הכתובת שהעתקתם.
עושים Redeploy לאפליקציה.
מרגע שזה מוגדר, הלידים נשמרים ישירות בטאב Leads בגוגל שיטס, והבדיקה לכפילות נעשית שם לפי טלפון או מייל.
