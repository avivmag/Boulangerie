# אפליקציית הטבת קינוחים

אפליקציה נקייה למסעדה: לקוח סורק QR, ממלא שם מלא, טלפון ומייל, מאשר דיוור אם הוא רוצה, ומקבל קוד חד-פעמי ל-2 קינוחים במתנה.

## הפעלה

```bash
npm start
```

פותחים בדפדפן:

```text
http://localhost:3000
```

## קובץ מעקב לאקסל

כל הרשמה נשמרת כאן:

```text
data/leads.csv
```

אפשר לפתוח את הקובץ ישירות באקסל. יש גם הורדה דרך:

```text
http://localhost:3000/api/export
```

## מניעת כפילויות

המערכת לא מאפשרת לממש שוב אם אותו טלפון או אותו מייל כבר קיימים. הנתונים הפנימיים נשמרים כאן:

```text
data/claims.json
```

## התאמה למסעדה

כדי להחליף לוגו, אפשר להוסיף קובץ תמונה לתיקיית `public` ולעדכן את אזור `brand-mark` ב-`public/index.html`.

## העלאה לאינטרנט

האפליקציה כוללת שרת ששומר הרשמות ומונע כפילויות, ולכן לא מתאימה ל-GitHub Pages בלבד.

הדרך הפשוטה:

1. מעלים את כל התיקייה ל-GitHub Repository.
2. פותחים חשבון ב-Render.
3. בוחרים New Web Service.
4. מחברים את ה-Repository.
5. Render יזהה את `render.yaml` ויריץ את האפליקציה.
6. אחרי הפריסה מקבלים כתובת ציבורית שאפשר להפוך ל-QR.

הערה חשובה: שמירה לקובץ CSV מתאימה לפיילוט. לקמפיין פעיל מומלץ לחבר Google Sheets או מסד נתונים כדי שהלידים יהיו שמורים גם אחרי פריסות ועדכונים.

## חיבור ל-Google Sheets

1. יוצרים Google Sheet חדש.
2. בתפריט בוחרים Extensions ואז Apps Script.
3. מדביקים את הקוד מהקובץ `google-apps-script.js`.
4. לוחצים Deploy ואז New deployment.
5. בוחרים Web app.
6. תחת Execute as בוחרים Me.
7. תחת Who has access בוחרים Anyone.
8. לוחצים Deploy ומאשרים הרשאות.
9. מעתיקים את כתובת ה-Web app שמסתיימת ב-`/exec`.
10. ב-Render מוסיפים Environment Variable בשם `GOOGLE_SHEETS_WEBHOOK_URL` עם הכתובת שהעתקתם.
11. עושים Redeploy לאפליקציה.

מרגע שזה מוגדר, הלידים נשמרים ישירות בטאב `Leads` בגוגל שיטס, והבדיקה לכפילות נעשית שם לפי טלפון או מייל.
